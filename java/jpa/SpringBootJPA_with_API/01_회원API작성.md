v1

API 제공을 위한 코드 작성

API 호출 시, Entity를 그대로 요청객체로 사용해보기

Entity를 수정했을 때, API의 스펙이 변경되는 위험

API호출을 위한 @Entity객체를 직접 사용하면 문제가 발생하는 경우가 많아

API 스펙에 맞춰 별도의 DTO객체를 생성하는 것이 좋다.



v2

API 요청을 받기 위한 dto클래스를 작성하여, 데이터를 받는다.

Entity의 스펙 변경에 따라서 API 인터페이스가 변경되지 않는다.

API스펙을 볼려면, v1은 엔티티를 봤을 때 한번에 알 수 없다. 하지만 dto를 보면 dto에만 맞게 데이터를 받을 수 있기 때문에 훨씬 좋다.

실무에서 엔티티를 외부에 노출하면,... 힘들다.
